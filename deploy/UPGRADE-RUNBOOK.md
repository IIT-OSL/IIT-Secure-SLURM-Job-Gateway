# UPGRADE-RUNBOOK.md — Manual Steps for feature/cluster-upgrades

> This file consolidates every `[GPU-HOST]` and `[LOGIN]` manual step required
> to activate the Phase 1–7 changes. Work through phases in order.
> **Do not run root commands on either node until the branch is merged and tests pass.**

---

## Phase 1 — GPU Software Stack

### [GPU-HOST] Install build-essential (unlocks torch.compile / Triton)

```bash
sudo bash /path/to/deploy/setup-compute-toolchain.sh
```

**Status:** Script is idempotent — safe to re-run.

---

## Phase 2 — SLURM Accounting (slurmdbd)

### [LOGIN] Full setup

Follow `deploy/setup-slurmdbd.md` in order:
1. Install MariaDB and create `slurm_acct_db`
2. Install `slurmdbd`, place `deploy/slurmdbd.conf` at `/etc/slurm/slurmdbd.conf`
3. Add accounting directives to `slurm.conf` on both nodes
4. Add QOS + per-user GPU limit via `sacctmgr`
5. Restart `slurmctld` / `slurmd`

After completion the TUI will auto-detect `sacct` and use it for job history.
Set `SACCT_ENABLED=0` in `/usr/local/bin/iit-gpu-manager` to force file-scan fallback.

## Phase 3 — Systemd GPU Stats Writer

### [GPU-HOST] Install and enable the stats writer service

```bash
# Copy writer and service from the login node
scp slurmadmin@192.168.122.10:/home/slurmadmin/IIT-Secure-SLURM-Job-Gateway/deploy/iit-gpu-stats-writer \
    /usr/local/bin/iit-gpu-stats-writer
chmod +x /usr/local/bin/iit-gpu-stats-writer

scp slurmadmin@192.168.122.10:/home/slurmadmin/IIT-Secure-SLURM-Job-Gateway/deploy/iit-gpu-stats.service \
    /etc/systemd/system/iit-gpu-stats.service

# Enable and start
sudo systemctl daemon-reload
sudo systemctl enable --now iit-gpu-stats

# Verify
sudo systemctl status iit-gpu-stats
```

After ~5 seconds verify the stats file is fresh:
```bash
ls -la /shared/.gpu_stats.json
python3 -c "import json; d=json.load(open('/shared/.gpu_stats.json')); print(d)"
```

### [GPU-HOST] Remove the old @reboot cron entry (if present)

```bash
crontab -l | grep -v "iit-gpu-stats-writer" | crontab -
```

### [GPU-HOST] Test crash recovery

```bash
sudo systemctl kill iit-gpu-stats
sleep 5
sudo systemctl status iit-gpu-stats   # should show Active: active (running)
```

## Phase 4 — Apptainer Container Support

### [GPU-HOST] Install Apptainer

```bash
sudo apt-get install -y software-properties-common
sudo add-apt-repository -y ppa:apptainer/ppa
sudo apt-get update && sudo apt-get install -y apptainer
apptainer --version
```

### [LOGIN] Install Apptainer (for building images)

```bash
sudo apt-get install -y apptainer
```

### [GPU-HOST] or [LOGIN] Create images directory

```bash
sudo mkdir -p /shared/images
sudo chmod 0775 /shared/images
# If gpuusers group exists:
# sudo chown root:gpuusers /shared/images
```

### [GPU-HOST] Build prebuilt images (see deploy/build-images.md)

```bash
cd /home/slurmadmin/IIT-Secure-SLURM-Job-Gateway
for img in llm-finetune llm-serve vision diffusion data-science; do
    sudo apptainer build /shared/images/${img}.sif deploy/images/${img}.def
done
```

## Phase 6 — Prebuilt Environments

### [LOGIN] Install conda specs as shared environments

Via the TUI: **Setup → Install a prebuilt environment** (runs as `slurmadmin`).

Or manually:
```bash
CONDA=/shared/miniforge3/bin/conda
for name in llm-finetune llm-serve vision diffusion data-science; do
    $CONDA env create -p /shared/envs/${name} \
        -f /home/slurmadmin/IIT-Secure-SLURM-Job-Gateway/envs/specs/${name}.yml \
        --force
done
```

## Phase 7 — Hardening

### [LOGIN+GPU-HOST] Create gpuusers group (for 0o770 job dirs)

```bash
# On BOTH nodes with the same GID:
sudo groupadd -g 2000 gpuusers
sudo usermod -aG gpuusers daham
sudo usermod -aG gpuusers slurm     # so slurmstepd can write job output

# Existing /shared/jobs dirs — fix permissions:
sudo find /shared/jobs -type d -exec chmod 0770 {} \;
sudo find /shared/jobs -type d -exec chgrp gpuusers {} \;
```

### [LOGIN] Disk quota runbook (XFS project quotas — optional)

```bash
# Requires /mnt/nvme_storage to be mounted with pquota option in /etc/fstab:
# /dev/nvme0n1p1 /mnt/nvme_storage xfs defaults,pquota 0 0
#
# Set per-user quota (e.g. 500 GB):
sudo xfs_quota -x -c 'project -s -p /shared/daham 100' /mnt/nvme_storage
sudo xfs_quota -x -c 'limit -p bsoft=450g bhard=500g 100' /mnt/nvme_storage
```
